How to use this in OBS Studio

A Import the scene file into OBS
  1. Download a scene file below (your saved scene, or a default template — they appear with download buttons).
  2. Open OBS Studio (download free at obsproject.com if you don't have it).
  3. In the OBS top menu, click Scene Collection → Import….
  4. In the dialog, click the … next to "Collection Path", browse to your downloaded .json, and click OK.
  5. Switch to the imported collection via Scene Collection menu (it appears in the list with the name "BambuBoard 3.0.0" or similar).
  6. Your scene is now active — widgets should populate with live data within a few seconds.
B Start the printer camera stream (Bambu Studio)

Bambu Lab's official way to feed the printer camera into OBS is through Bambu Studio's virtual camera, which writes an RTP stream that OBS reads via a Media Source (next step). Note: currently supported on X1, X1C, X1E. H2D, H2S, and P2S are not yet supported by Bambu Lab's virtual camera per their wiki — for those, use a direct RTSP source instead (see notes at bottom).

  1. Open Bambu Studio and go to the Device tab.
  2. Click the Camera Settings gear icon (top-right of the camera panel).
  3. Toggle Go Live ON. First time only: Bambu Studio prompts you to install Virtual Camera Tools — accept and let it install.
  4. Closing Bambu Studio doesn't stop the stream — the virtual camera tool keeps running in the background.
C Add a Media Source in OBS for the camera feed

The downloaded BambuBoard scene includes all the data widgets (temps, AMS, fans, progress) but not the camera — that's a Media Source you add manually because it points at a file on your machine.

  1. In OBS, in the Sources panel, click + → Media Source. Name it Printer Camera (or anything you like). Click OK.
  2. In the Media Source properties dialog:
    • Uncheck "Local File" (OBS handles SDP files better via the file: URL path).
    • Input: paste the path to ffmpeg.sdp with a file: prefix:
      • Windows: file:C:/Users/<you>/AppData/Roaming/BambuStudio/cameratools/ffmpeg.sdp
      • macOS: file:/Users/<you>/Library/Application Support/BambuStudio/cameratools/ffmpeg.sdp
    • FFmpeg Options: protocol_whitelist=file,udp,rtp (this is required — without it, OBS refuses to read RTP from a file).
    • Click OK. The camera preview should appear in the source.
  3. Position / resize the Printer Camera source under the BambuBoard widgets so it sits behind them in the layout (or wherever you prefer).

Full Bambu Lab guide with screenshots: Bambu Lab Wiki — Virtual Camera

H2D / H2S / P2S users: Bambu Studio's virtual camera doesn't support these models yet. Workaround: enable LAN Only Liveview on the printer (Settings → Network), then in OBS use a Media Source with input rtsps://bblp:<ACCESS_CODE>@<PRINTER_IP>:322/streaming/live/1 and the same FFmpeg options — see Bambu Lab Support for firmware requirements.

D Keep BambuBoard running while streaming

Each widget in OBS is a Browser Source pointing at this BambuBoard server. The server must be running for widgets to receive live data — leave it running on the same machine (or a Pi / always-on host on your network) for the duration of your stream.
